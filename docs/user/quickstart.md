# Quick Start

## Python

```python
from py_agent_ctrl import AgentCtrl

response = AgentCtrl.claude_code().execute("Summarize this repository.")
print(response.text)
```

## CLI

```bash
uv run ctrlagent execute --agent claude-code "Summarize this repository."
```

## Common Builder Methods

- `.with_model(...)`
- `.with_system_prompt(...)`
- `.append_system_prompt(...)`
- `.with_max_turns(...)`
- `.in_directory(...)`
- `.with_additional_dirs([...])`
- `.with_timeout(...)`
- `.with_sandbox_driver(...)`
- `.resume_session(...)`
- `.continue_session()`
- `.on_text(...)`
- `.on_tool_call(...)`
- `.on_event(...)`
- `.on_complete(...)`
- `.on_error(...)`

Use `AgentCtrl.make("codex")` or `AgentCtrl.make(AgentType.CODEX)` when the agent type is selected at runtime.

## Stream Events

The stable event classes are `AgentTextEvent`, `AgentToolCallEvent`,
`AgentResultEvent`, and `AgentUnknownEvent`. Providers can also emit richer
events such as `AgentReasoningEvent`, `AgentPlanUpdateEvent`,
`AgentUsageEvent`, `AgentWarningEvent`, and `AgentFileChangeEvent` when their
native stream includes that information.

## Sessions

`AgentResponse.execution_id` is generated by `py-agent-ctrl` for each execution.
`AgentResponse.session_id` is the provider session/thread ID returned by the
agent CLI, when available.

Session helpers expose `SessionCapabilities`, `SessionInfo`, and
`SessionOperation` for callers that need to reason about resume/continue support
without assuming every provider supports listing, loading, forking, tagging,
renaming, or deleting sessions.

## Permissions

Permission support is optional and provider-dependent. The shared models are
`PermissionRequest`, `PermissionOption`, `PermissionResponse`,
`PermissionOptionKind`, and `PermissionOutcome`. `PermissionBroker` provides a
small pending-request store, available from `py_agent_ctrl.actions`, for
integrations that need to resolve or cancel permission requests.

## Agent-Specific Methods

- Claude Code: `.with_permission_mode(...)`, `.with_allowed_tools(...)`
- Codex: `.with_sandbox(...)`, `.full_auto()`, `.dangerously_bypass()`, `.with_images([...])`
- OpenCode: `.with_agent(...)`, `.with_files([...])`, `.with_title(...)`, `.share_session()`
- Pi: `.with_provider(...)`, `.with_thinking(...)`, `.with_tools([...])`, `.with_extensions([...])`, `.with_skills([...])`
- Gemini: `.with_approval_mode(...)`, `.plan_mode()`, `.yolo()`, `.with_sandbox()`, `.with_allowed_tools([...])`

Provider-native mode helpers accept either strings or typed enums such as
`ClaudePermissionMode`, `CodexSandboxMode`, and `GeminiApprovalMode`. Invalid
mode values fail early.
